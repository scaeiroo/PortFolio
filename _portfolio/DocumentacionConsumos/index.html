Introducción al Proyecto

Comprensión del Sistema de Monitoreo de Placas Solares

Para iniciar este proyecto, fue esencial entender cómo funciona el sistema de monitoreo de placas solares implementado en Lacor. El sistema actual permite el acceso a datos de consumo energético a través de una aplicación web, pero presenta limitaciones en la extracción automática de informes y su posterior análisis. La observación detallada de los datos disponibles, su formato y el método manual de extracción fueron clave para diseñar una solución eficiente que automatice este proceso.

Problemática Identificada

La primera fase consistió en crear una máquina virtual con Ubuntu 24.04.1. En ella, instalé El principal desafío identificado fue la imposibilidad de extraer de manera automatizada todos los datos necesarios utilizando las librerías estándar de Python. Las API disponibles no proporcionaban la totalidad de la información que sí está accesible mediante la descarga manual desde la interfaz web. Esta limitación obligaba a un proceso manual y tedioso para la obtención de los informes diarios, mensuales y anuales, dificultando el análisis continuo del rendimiento de las placas solares.

Los intentos iniciales con diversas librerías de Python revelaron que los datos obtenidos no coincidían en cantidad ni calidad con los descargables manualmente desde la plataforma, lo que hizo necesario replantear la estrategia de extracción.

Objetivos del Proyecto

El proyecto se estructura en torno a dos objetivos principales:

1- Desarrollo de una aplicación de escritorio que permita:

  - Extracción automatizada de informes diarios, mensuales y anuales
  - Visualización gráfica de los datos obtenidos
  - Interfaz intuitiva para la interacción con los datos

2- Implementación de un script de automatización que:

  - Ejecute diariamente la extracción de datos
  - Almacene la información en una base de datos MySQL
  - Garantice la continuidad del registro histórico

Configuración del Entorno de Desarrollo

Para abordar este proyecto, se configuró un entorno de desarrollo con las siguientes herramientas:

  • Python 3.10: Lenguaje principal para el desarrollo del scraping y la aplicación

  • Selenium: Framework para automatizar la navegación web y extracción de datos

  • MySQL: Sistema de gestión de base de datos para el almacenamiento permanente

  • PyQt5/Tkinter: Bibliotecas para el desarrollo de la interfaz gráfica

  • Matplotlib/Seaborn: Herramientas para la visualización gráfica de datos

  • GitHub: Sistema de control de versiones para el seguimiento del desarrollo

Este conjunto de herramientas proporciona la base tecnológica necesaria para desarrollar una solución robusta que cumpla con los requisitos establecidos. La elección de Selenium como herramienta de web scraping responde directamente a la necesidad de emular la navegación humana para acceder a todos los datos disponibles en la plataforma.

Metodología de Trabajo

El desarrollo del proyecto sigue una metodología iterativa:

1- Análisis preliminar: Estudio de la estructura de la web y los datos disponibles

2- Desarrollo del scraper: Implementación del sistema de extracción automatizada

3- Construcción de la aplicación: Desarrollo de la interfaz y funcionalidades de usuario

4- Automatización: Creación del script para la ejecución programada

5- Pruebas y optimización: Verificación de la fiabilidad y rendimiento del sistema

Esta aproximación garantiza la adaptabilidad del sistema ante posibles cambios en la plataforma de origen, manteniendo la integridad de los datos y la continuidad del servicio.
