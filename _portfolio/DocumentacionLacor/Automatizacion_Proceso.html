Automatización del Proceso

Estos scripts están diseñados para gestionar las conexiones necesarias y ejecutar de forma automatizada las funciones del script XX_LLAMADA_PRODUCCION.R, garantizando una actualización eficiente de los datos.

Automatización de la Carga de Datos

El script XX_LLAMADA_PRODUCCION.R se ejecuta automáticamente en segundo plano cada día a las siguientes horas;

-6:00 hasta las 22:00 cada 15 mins

La tarea programada en cron se configura en el sistema/servidor de Ubuntu editando el crontab con:

crontab -e

e insertando la siguiente línea:

# === Actualizacion Produccion Lacor === #

# Este Script se ejecutara cada **15 minutos**, desde las **05:00 hasta las 22:45**, en las siguientes horas: **05:00, 05:15, 05:30, 05:45, ... hasta las 22:45**.  

*/15 5-22 * * * cd /home/dataml/Documentos/GitHub/Lacor && /usr/bin/Rscript Scripts/XX_LLAMADA_PRODUCCION.R

Además, es necesario otorgar permisos de ejecución al script con:

chmod +x /home/scaeiro/GIT/Lacor/Scripts/XX_LLAMADA_PRODUCCION.R

Asi nos aseguramos una carga de datos continua y sin intervención manual. Esta configuración optimiza el flujo de trabajo y mantiene la información siempre actualizada.

Automatizacion del Script:

El proceso de ejecución automática del script sigue los siguientes pasos:

  • Carga de configuración y funciones: Se importan los archivos de configuración y las funciones necesarias.

  • Actualización desde GitHub: Se ejecuta PullGitHub() para sincronizar el código.

  • Conexión a bases de datos: Se establecen conexiones con SQL Server y MySQL.

  • Consulta y procesamiento de datos: Se ejecutan consultas SQL y se realiza la limpieza y transformación de datos.

  • Carga de datos a MySQL: Los datos procesados se almacenan en la base de datos MySQL.

  • Liberación de recursos: Se desconectan las bases de datos y se limpia el entorno de R.
