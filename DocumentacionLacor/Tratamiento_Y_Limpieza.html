Tratamiento y Limpieza de Datos

2_Mantener_DatosActualizados.R

Carga y Actualización de Datos:

Este script está diseñado para actualizar y fusionar datos de producción, asegurando que los archivos se mantengan al día con la información más reciente.

  1. Creacio del RData Diario Antes de iniciar el proceso sacamos los datos del mismo dia y cerramos conexiones ya que la consulta ya esta realizada.

      #===========================================================#
  #         CREAR RDATA DIARIO Y CERRAR CONEXIONES            #
  #===========================================================#
  Actualizar_Datos_ProduccionDiario(conn)
  # Desconexion SQLServer
  dbDisconnectAllSQLServer()

  2. Cargar los DataFrames: El primer paso es cargar los archivos de producción diaria y total para el día actual. Usamos Sys.Date() para obtener la fecha actual y luego cargamos los archivos correspondientes.

      # Obtener la fecha actual 📅
  fecha_actual <- Sys.Date()

  archivo_diario <- paste0("./Output/Produccion_Diario_DIA_", fecha_actual, ".RData")
  archivo_total <- "./Output/Produccion_TOTAL.RData"

  cat(paste("🔄 Iniciando la Fusión de Datos para actualizar el archivo:\n"))
  DF_ProduccionDiario <- CargarRData(archivo_diario)
  DF_Produccion_TOTAL <- CargarRData(archivo_total)

  3. Fusión de Datos: Una vez cargados los datos, se verifica la estructura de los DataFrames y se ajustan las columnas para garantizar que ambos DataFrames tengan el mismo formato. Esto se hace eliminando la última columna del DataFrame diario y renombrando las columnas para que coincidan con el DataFrame total.

    # Verificación de la estructura de los datos y renombrado de columnas
columnas_Produccion_diario_df <- colnames(ProduccionDiarioDF)
columnas_Produccion_total_df <- colnames(ProduccionDiario_TOTAL_DF)
ProduccionDiarioDF <- ProduccionDiarioDF[, -ncol(ProduccionDiarioDF)]
names(ProduccionDiarioDF) <- columnas_Produccion_total_df

  4. Eliminación de Duplicados: Después de fusionar los datos, se eliminan posibles filas duplicadas utilizando la función FiltrarDuplicadosTOTALDIARIO para garantizar que los datos sean únicos y no haya redundancias.

    # Eliminación de filas duplicadas
ProduccionDiario_TOTAL_DF <- FiltrarDuplicadosTOTALDIARIO(ProduccionDiario_TOTAL_DF, ProduccionDiarioDF)

  5. Guardar y Sobrescribir los Datos: Finalmente, los datos fusionados y limpios se guardan en los archivos correspondientes, tanto en formato .RData como en .CSV para su posterior uso y análisis.

    # Guardar archivos actualizados
GuardarRData(ProduccionDiarioDF, archivo_diario)
GuardarRData(ProduccionDiario_TOTAL_DF, archivo_total)
GuardarCSV(ProduccionDiario_TOTAL_DF, "Produccion_Diario_TOTAL", "./Output/CSV")

3_Trasformacion_Y_Limpieza.R

Transformaciones y Limpiezas de Datos:

Este script realiza la transformación y limpieza de varios DataFrames relacionados con los datos tanto de Producción como lo Planificado.

Carga de Datos:

El primer paso consiste en cargar los datos de las diferentes fuentes, incluyendo archivos .RData y .CSV. Esto proporciona los datos necesarios para realizar las transformaciones posteriores.

cat(paste("🔄 Iniciando la Limpieza de Datos de OLANET:\n"))
DF_CANTIDAD <-CargarRData("./Output/DatosPrevistos/Datos_OF_Cantidad.RData")
DF_PRODUCIDO <- CargarCSV("./Output/CSV/Produccion_TOTAL.csv")
DF_PLANIFICADO <- CargarCSV("./Output/CSV/Olanet_Planificado.csv")

Tranformaciones y Limpieza:

  1. Procesamiento de Datos Planificados con Cantidades: Se fusiona el DataFrame DF_PLANIFICADO con la información de cantidades previstas de DF_CANTIDAD. Esto permite enriquecer los datos planificados con información sobre las cantidades que se esperan producir.

           DF_PLANIFICADO_CANTIDAD <- merge(DF_PLANIFICADO, DF_CANTIDAD[, c("smOrdenId", "SmfaseId", "smMaterialId", "smMaterialnme", "CantidadaProducirnbr")], 
                                    by.x = c("OF", "FASE"), 
                                    by.y = c("smOrdenId", "SmfaseId"), 
                                    all.x = TRUE)

    colnames(DF_PLANIFICADO_CANTIDAD)[colnames(DF_PLANIFICADO_CANTIDAD) == "CantidadaProducirnbr"] <- "CantidadPrevista"
    DF_PLANIFICADO_CANTIDAD <- DF_PLANIFICADO_CANTIDAD[, -c(ncol(DF_PLANIFICADO_CANTIDAD)-1, ncol(DF_PLANIFICADO_CANTIDAD)-2)]

  2. Filtrado de Datos Planificados: Se aplica un filtro para eliminar referencias que comienzan con “MTO” o que contienen espacios, lo que asegura que solo se consideren referencias válidas en el análisis.

       DF_PLANIFICADO_CANTIDAD <- DF_PLANIFICADO_CANTIDAD %>%
  filter(!grepl("^MTO", REFERENCIA) & !grepl("\\s", REFERENCIA))

  3. Limpieza de Datos de Producción: Se procesan los datos de producción, asegurando que el campo OF contiene solo valores numéricos válidos y filtrando aquellos que empiezan con “24-”. Además, se eliminan las filas que no tienen valor en MATERIAL_SALIDA.

      DF_PRODUCIDO <- DF_PRODUCIDO %>%
  filter(!is.na(OF) & grepl("^[0-9]+$", OF)) %>%
  mutate(OF = as.numeric(OF)) %>%
  filter(!grepl("^24-", OF))
DF_PRODUCIDO <- DF_PRODUCIDO[!is.na(DF_PRODUCIDO$MATERIAL_SALIDA), ]

Guardar los Archivos Resultantes:

Después de realizar todas las transformaciones y limpiezas necesarias, los DataFrames resultantes se guardan en archivos .CSV para su posterior análisis y uso.

# Guardar los DataFrames resultantes en CSV
GuardarCSV(DF_PLANIFICADO_CANTIDAD,"Olanet_Planificado","./OlanetDatos")
GuardarCSV(DF_PRODUCIDO,"Olanet_Producido","./OlanetDatos")

Resumen Final

En resumen, este conjunto de scripts realiza las siguientes operaciones:

  • Carga de datos de diferentes fuentes.
  • Fusión de datos, eliminación de duplicados y valores no validos
  • Guardado de los datos procesados en archivos .CSV para su uso posterior.
