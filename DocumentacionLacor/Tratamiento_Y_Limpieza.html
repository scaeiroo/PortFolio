2_Mantener_DatosActualizados.R

  1. Creacio del RData Diario Antes de iniciar el proceso sacamos los datos del mismo dia y cerramos conexiones ya que la consulta ya esta realizada.

  #         CREAR RDATA DIARIO Y CERRAR CONEXIONES            #
  #===========================================================#
  Actualizar_Datos_ProduccionDiario(conn)
  # Desconexion SQLServer
  dbDisconnectAllSQLServer()

  2. Cargar los DataFrames: El primer paso es cargar los archivos de producci√≥n diaria y total para el d√≠a actual. Usamos Sys.Date() para obtener la fecha actual y luego cargamos los archivos correspondientes.

  cat(paste("üîÑ Iniciando la Fusi√≥n de Datos para actualizar el archivo:\n"))

  DF_Produccion_TOTAL <- CargarRData(archivo_total)

  3. Fusi√≥n de Datos: Una vez cargados los datos, se verifica la estructura de los DataFrames y se ajustan las columnas para garantizar que ambos DataFrames tengan el mismo formato. Esto se hace eliminando la √∫ltima columna del DataFrame diario y renombrando las columnas para que coincidan con el DataFrame total.

columnas_Produccion_total_df <- colnames(ProduccionDiario_TOTAL_DF)

names(ProduccionDiarioDF) <- columnas_Produccion_total_df

  4. Eliminaci√≥n de Duplicados: Despu√©s de fusionar los datos, se eliminan posibles filas duplicadas utilizando la funci√≥n FiltrarDuplicadosTOTALDIARIO para garantizar que los datos sean √∫nicos y no haya redundancias.

Transformaciones y Limpiezas de Datos:

Este script realiza la transformaci√≥n y limpieza de varios DataFrames relacionados con los datos tanto de Producci√≥n como lo Planificado.

Carga de Datos:

El primer paso consiste en cargar los datos de las diferentes fuentes, incluyendo archivos .RData y .CSV. Esto proporciona los datos necesarios para realizar las transformaciones posteriores.

  1. Procesamiento de Datos Planificados con Cantidades: Se fusiona el DataFrame DF_PLANIFICADO con la informaci√≥n de cantidades previstas de DF_CANTIDAD. Esto permite enriquecer los datos planificados con informaci√≥n sobre las cantidades que se esperan producir.

  2. Filtrado de Datos Planificados: Se aplica un filtro para eliminar referencias que comienzan con ‚ÄúMTO‚Äù o que contienen espacios, lo que asegura que solo se consideren referencias v√°lidas en el an√°lisis.

  filter(!grepl("^MTO", REFERENCIA) & !grepl("\\s", REFERENCIA))

  3. Limpieza de Datos de Producci√≥n: Se procesan los datos de producci√≥n, asegurando que el campo OF contiene solo valores num√©ricos v√°lidos y filtrando aquellos que empiezan con ‚Äú24-‚Äù. Adem√°s, se eliminan las filas que no tienen valor en MATERIAL_SALIDA.

  filter(!is.na(OF) & grepl("^[0-9]+$", OF)) %>%
  mutate(OF = as.numeric(OF)) %>%
  filter(!grepl("^24-", OF))
DF_PRODUCIDO <- DF_PRODUCIDO[!is.na(DF_PRODUCIDO$MATERIAL_SALIDA), ]
